# 1 Flexible Uplink Processing Overview

The feature simplifies PUCCH

configuration and allows the configuration of PPPs.

| Feature Name           | Flexible Uplink Processing             |
|------------------------|----------------------------------------|
| Feature Identity       | FAJ 121 5155                           |
| Value Package Name     | LTE Base Package                       |
| Value Package Identity | FAJ 801 0400                           |
| Node Type              | Baseband Radio Node                    |
| Access Type            | LTE                                    |
| Licensing              | Basic feature. No license is required. |

Summary

The number of possible scheduling requests and CQI users are calculated based on the

number of PPPs configured. The Flexible Uplink Processing feature increases the PPPs

through configurable parameters.

The improved PPP capacity in a cell results in increased number of supported

scheduling requests and CQI users on cell level.

The feature improves flexibility in distribution of scheduling request resources

among cells while supporting maximum connected users capacity.

# 3 Feature Operation of Flexible Uplink Processing

The feature increases processing load and potential impact on PUSCH by using the physical

layer processing cycle for PUCCH and PUSCH trading. It introduces the following

parameters to define the number of PUCCH Format 1 and Format 2 PPPs per cell:

- EUtranCellFDD.noOfPucchFormat1PrbPairsPerFrameConf
- EUtranCellTDD.noOfPucchFormat1PrbPairsPerFrameConf
- EUtranCellFDD.noOfPucchFormat2PrbPairsPerFrameConf
- EUtranCellTDD.noOfPucchFormat2PrbPairsPerFrameConf

# 4 Network Impact of Flexible Uplink Processing

The feature can be

configured to improve performance in high load conditions.

Capacity and Performance

The feature increases PPPs for Baseband and improves performance in high load

conditions.

Increased PPPs increase the physical layer processing demand of PUCCH and that can

have an impact on the PUSCH capacity because of trading.

The increased PPP capacity in a cell can increase the number of supported scheduling

requests and CQI users on cell level.

Interfaces

No impact.

Other Network Elements

No impact.

# 8 Engineering Guidelines for Flexible Uplink Processing

The feature can be associated with several configuration parameters for the optimized

operation and network performance.

PUSCH and PUCCH Interference Avoidance

To avoid PUSCH interference with PUCCH, it is recommended to use the same number of

PUCCH RBs in all cells on the same frequency layer. To do this, set the

noOfPucchFormat1PrbPairsPerFrameConf and

noOfPucchFormat2PrbPairsPerFrameConf attributes belonging to

the EUtranCellTDD and EUtranCellFDD MOs in all

cells to the same frequency layer.

Primary Cell in Carrier Aggregation

A cell is considered as a primary cell in Carrier Aggregation, if the following is

true:

- Carrier Aggregation is enabled
- Dynamic SCell Selection is enabled
- noOfChannelSelectionSets &gt; 0
- dlChannelBandwidth  5000

## 8.1 Enable Flexible Uplink Processing

The Flexible Uplink Processing feature can be enabled through a configurable

parameter.

The feature can be enabled by default after the Automatic Flexible Uplink Processing

feature enhancement is installed. If the

ENodeBFunction.ulPhyProcResTradingEnabled attribute is set to

a non-zero value, the system automatically changes it to 1. If

ENodeBFunction.ulPhyProcResTradingEnabled is set to a default

value, the system automatically changes it to 2.

Prerequisites

The following features are disabled:

- Basic AAS for FDD
- Basic AAS for TDD

Steps

1. Lock all the cells on the eNodeB.
2. Set the ENodeBFunction.ulPhyProcResTradingEnabled attribute to 1 or 2.
3. Unlock all the cells on the eNodeB.

## 8.2 Disable Flexible Uplink Processing

The feature can be disabled through a configurable attribute.

Prerequisites

The following features are disabled:

- Basic AAS for FDD
- Basic AAS for TDD

Steps

1. Lock all the cells on the eNodeB.
2. Set the ENodeBFunction.ulPhyProcResTradingEnabled parameter to 0.
3. Unlock all the cells on the eNodeB.
