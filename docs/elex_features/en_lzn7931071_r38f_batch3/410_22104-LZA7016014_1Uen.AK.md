# 1 Support for AISG RAE Overview

The feature adds support for remote control and monitoring capabilities that are added

to the Antenna Line Devices (ALD) standard published by the AISG.

Summary

The RAE is a subsidiary module that enables to retrieve the following parameters about the ALD

remotely from the antenna:

- Altitude
- Latitude
- Longitude
- Downtilt
- Azimuth
- Roll

For the desired beam pattern, the radio must be configured with the right weighting

factor. The right weighting factor configuration can be validated with these

parameters.

Support for AISG RAE version 2.2.0 extends the geographical location information and antenna

observability functions to the ASD and GLS subunits. The ASD and GLS subunits allow

for benchmarking these parameters, and provide the geographical location information

of the ALD.

The RAE can upload the collected information to ENM.

The Support of AISG RAE feature has the following benefits:

- Improves the observability of remote antenna parameters
- Improves network optimization possibilities by tracking and collecting antenna device information
- Provides data for beamforming through the antenna parameters
- Reduces time cost and labor cost of tower maintenance
- Provides support for the ASD and GLS subunits

# 2 Dependencies of Support for AISG RAE

Feature Dependencies

There are no dependencies to other features.

Hardware

This feature requires AISG 2.0 RAE or higher.

The feature is supported on the following Hardware:

- Massive MIMO radio units with AISG port for RAE subunits.
- Radio units with AISG ports for ASD and GLS subunits.

Limitations

The action RaeSubUnit.downloadAntennaPatternFile requires one

download at a time. Starting a new download while

RaeSubUnit.downloadAntennaPatternFile is still running result

in download failure of both actions.

Network Requirements

No network requirements for this feature.

# 3 Feature Operation of Support for AISG RAE

AISG RAE can be integrated into the smart antenna, or it can be used as the external

device.

Support for AISG RAE enables to perform the following actions:

- Registering, configuring and upgrading RAE, GLS or ASD remotely
- Displaying information about the weighting factor file of each RAE and about the capabilities of each RAE, GLS or ASD
- Checking data observed by RAE, GLS or ASD
- Uploading beam weighting files remotely to RAE
- Scanning RAE, RET, GLS or ASD units for each hardware
- Scanning cascaded RAE, GLS or ASD units for each hardware
- Reading the number of RAE, GLS or ASD subunits
- Fault handling for AISG

Information collected about RAE can be stored in ENM.

Support for AISG RAE provides support for the ASD and GLS subunits, that provide the

following services:

- Geographical location information
- Observing the physical alignment of the ALD

# 7 Activate Non-License-Controlled Features

Non-license-controlled features do not require manual activation and cannot be

deactivated.

While license-controlled features need to be activated, non-license-controlled features

are activated automatically during node integration. They also do not

have

a software license key associated, which means that they have no

entries in the LKF that is installed on the node.

In some cases, non-license-controlled features need to be configured

manually or have some of

their

subfunctions enabled to work properly.

- AISG 2.2 RAE, ASD and GLS support for AAS-capable radio units.

requirements.
