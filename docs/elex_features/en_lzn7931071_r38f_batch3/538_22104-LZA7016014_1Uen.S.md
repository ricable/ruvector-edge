Summary

The Support for AISG ADB feature adds functionality support for the ADB subunit and enables

integration of the unit into the network. The ADB subunit allows data collection

related to antenna arrays.

# 3 Feature Operation of Support for AISG ADB

This feature supports for AISG ADB and related data

collection.

The system detects connected ADB subunits. When the ADB subunit is discovered in the

Antenna Line Device (ALD) subunit list, an AISG version is negotiated for ADB to find

the latest available version. The negotiated AISG version, which is supported by both

RBS and ALD, is set as configured for the ADB subunit.

When connection to AntennaNearUnit is established, the system creates

the AdbSubUnit MO under the AntennaNearUnit

MO.

ADB allows data collection by performing the following actions:

- Setting the AntennaNearUnit.antennaUnitRef attribute to update antenna related data on AntennaUnit MO
- Executing the AdbSubUnit.getLogicalAntennaArraysData action to collect logical antenna array related data
- Running AIGS device scan to collect ADB subunit related data into an XML file

The MALD is supported with AISG 3.0. If the MALD is configured, the ADB unit can be

connected to more than one primary node. For more information about MALD, see Manage Hardware Equipment.

# 7 Activate Non-License-Controlled Features

Non-license-controlled features do not require manual activation and cannot be

deactivated.

While license-controlled features need to be activated, non-license-controlled features

are activated automatically during node integration. They also do not

have

a software license key associated, which means that they have no

entries in the LKF that is installed on the node.

In some cases, non-license-controlled features need to be configured

manually or have some of

their

subfunctions enabled to work properly.

The feature enhancement introduces following actions:

- AntennaNearUnit.clearActiveAlarms
- AntennaNearUnit.maldGetSetup
- AntennaNearUnit.maldSetSetup
- AntennaNearUnit.maldResetSetup
- AdbSubUnit.clearActiveAlarms
- RetSubUnit.clearActiveAlarms
- TmaSubUnit.clearActiveAlarms

Interfaces

No impact.

Hardware

The feature enhancement is applicable to the following near ALDs compatible with AISG

3.0:

- ADB
- RET
