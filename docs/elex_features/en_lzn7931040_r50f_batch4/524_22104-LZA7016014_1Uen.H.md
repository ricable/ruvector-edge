# 1 LTE Persistent PRACH Configuration Overview

The feature enables the manual configuration and improves the automatic assignment of

the PRACH configuration index. The feature protects the PRACH configuration index from

unnecessary updates at cell lock and unlock.

| Feature Name           | LTE Persistent PRACH Configuration                     |
|------------------------|--------------------------------------------------------|
| Feature Identity       | FAJ 121 5303                                           |
| Value Package Name     | LTE Base Package                                       |
| Value Package Identity | FAJ 801 0400                                           |
| Node Type              | Baseband Radio Node                                    |
| Access Type            | LTE                                                    |
| Licensing              | Non-license-controlled feature. No license is required |

Summary

The LTE Persistent PRACH Configuration feature allows the operator to select the automatic or

manual setting of the PRACH Configuration Index used in the cell.

When automatic assignment is selected, the value of the PRACH Configuration Index can remain

unchanged or it can be reassigned by the system at cell lock and unlock. Automatic

assignment allows the index to be updated only when the preamble format is

changed.

When manual configuration is selected, the value of the PRACH Configuration Index is

set by the operator. The operator defines the value of the index by setting a

specific starting subframe for PRACH transmissions in the cell.

The feature provides the following benefits:

- The PRACH configuration index can remain persistent during cell lock and unlock.
- The operator can set a specific starting subframe for PRACH transmissions in the cell.
- The operator can observe the value of PRACH configuration index that is used in cell.

Additional Information

More information about the LTE Persistent PRACH Configuration feature and related topics can

be found in the following documentation:

- 3GPP TS 36.211; LTE; Evolved Universal Terrestrial Radio Access (E-UTRA); Physical Channels and Modulation

# 3 Feature Operation of LTE Persistent PRACH Configuration

The feature enables the manual configuration and improves the automatic assignment of

the PRACH configuration index.

The impact of the LTE Persistent PRACH Configuration feature can be enabled or disabled

by setting the ENodeBFunction.prachConfigEnabled attribute to

true or false.

When the feature is disabled, the value of the PRACH Configuration Index is assigned

automatically by the system during cell upgrade. In this scenario, the value of the

index can change each time the cell is locked and unlocked.

When the feature is enabled, the prachSfn parameter is automatically

validated at next cell unlock. The operator can enable the feature before or after

setting this parameter to the desired value.

The operator can use the prachSfn parameter to choose between the

following configuration modes:

If

the

Flexible Uplink Processing feature is enabled and the manually set PRACH configuration

index is not

well-distributed

among the cells for a node, it might lead to considerable reduction of the processing

resource for PUSCH and PUCCH. This applies to some TTIs which have too many PRACH

occasions.

# 8 Engineering Guidelines for LTE Persistent PRACH Configuration

The value of the PRACH configuration index is assigned based on the value of the

PRACH preamble format.

The operator can define the value of the PRACH configuration index by setting a specific

starting subframe for PRACH transmissions in the cell using the

EutranCellFDD.prachSfn or the

EutranCellTDD.prachSfn attribute. The value of these attributes

is assigned according to PRACH preamble format.

PRACH Configuration Index in FDD

The value of the preamble format is derived from the cell range in the following

way:

- PRACH preamble format 0 is used when the maximum cell range is 15 km.
- PRACH preamble format 1 is used when the cell range is larger than 15 km.

The operator can adjust the cell range with the help of the

EUtranCellFDD.cellRange attribute.

The value of the PRACH configuration index is assigned according to Table 3.

PRACH Configuration Index in TDD

The value of the preamble format is derived from the cell range and the uplink or the

downlink subframe assignment in the following way:

- PRACH preamble format 0 is used when either of the following conditions apply: The maximum cell range is 15 km. The cell range is larger than 15 km and subframe assignment 2 is used.
- PRACH preamble format 1 is used when the cell range is larger than 15 km and subframe assignment 1 is used.

The operator can adjust the cell range with the help of the

EUtranCellTDD.cellRange attribute. The subframe assignment

can be configured with the help of the

EUtranCellTDD.subframeAssignment attribute.

## 8.1 Configure LTE Persistent PRACH Configuration

To select the automatic assignment or manual configuration of the PRACH configuration

index, the operator can use the prachSfn parameter.

Prerequisites

The LTE Persistent PRACH Configuration feature is activated.

To validate the value of the prachSfn parameter, the feature

must be enabled. The operator can enable the feature before or after setting this

parameter to the desired value.

![Image](../images/524_22104-LZA7016014_1Uen.H/additional_3_CP.png)

Steps

1. Lock the cell.
2. Set the EutranCellFDD.prachSfn or the EutranCellTDD.prachSfn attribute to the desired value, depending on configuration mode: Configuration Mode Attribute Value Automatic assignment -1 Manual configuration Other than -1
3. Unlock the cell

Feature Operation of LTE Persistent PRACH Configuration

## 8.2 Enable LTE Persistent PRACH Configuration

The impact of the feature can be enabled with the

ENodeBFunction.prachConfigEnabled attribute.

Prerequisites

The LTE Persistent PRACH Configuration feature is activated.

Steps

1. Lock all eNodeB cells.
2. To enable the feature, set the ENodeBFunction.prachConfigEnabled attribute to true.
3. Unlock all eNodeB cells.
