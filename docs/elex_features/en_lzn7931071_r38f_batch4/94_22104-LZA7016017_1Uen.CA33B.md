# 1 Measurement Gap-Aware NR Scheduling Overview

This feature enables the Master Node and the Secondary Node in dual connectivity

configurations to coordinate the configuration of the measurement gap and to perform

measurements that require a measurement gap.

| Feature Name           | Measurement Gap-Aware NR Scheduling                     |
|------------------------|---------------------------------------------------------|
| Feature Identity       | FAJ 121 5295                                            |
| Value Package Name     | NR Base Package                                         |
| Value Package Identity | FAJ 801 4002                                            |
| Node Type              | Baseband Radio Node                                     |
| Access Type            | NR                                                      |
| Licensing              | Non-license-controlled feature. No license is required. |

Summary

In NR standalone, the feature enables the UE to perform measurements that require a

measurement gap. In EN-DC

and

NR-DC, the feature enables the Master Node and the Secondary Node

to coordinate the configuration of the measurement gap. The measurement gap is

necessary for UEs for inter-frequency and IRAT measurements.

The feature has the following benefits:

- In EN-DC and NR-DC, the Master Node and the Secondary Node can coordinate the configuration of the measurement gap.
- In NR standalone, the UE is able to perform measurements that require a measurement gap.
- In EN-DC and NR-DC, there is no DTX due to the configured measurement gap on MCG or on SCG radio resources. This is because the measurement gap on the Master Node is communicated to the Secondary Node.
- As the scheduler is aware of the measurement gap, the scheduler uses radio resources efficiently to the UEs that are not in the measurement gap.

Additional Information

For more information on the RRC protocol, see 3GPP TS 38.331; NR; Radio

Resource Control (RRC); Protocol specification.

# 2 Dependencies of Measurement Gap-Aware NR Scheduling

The feature requires the Basic Intelligent Connectivity or the NR-NR Dual

Connectivity feature to be activated to achieve EN-DC or NR-DC functionality, respectively.

The feature is also operable in NR standalone deployments.

Hardware Requirements

Measurements with 3 ms measurement gaps are only applicable to Low-Band and Mid-Band

configurations.

Limitations

In NR standalone deployments, the following limitations apply:

- The measurement gap requirement information reported by the UE is limited by 3GPP. The measurement gap configuration for IRAT measurements is not considered.
- Measurement gaps for EPS fallback measurements might overlap with the NR TRS for Low-Band cells or the periodic SRS for Mid-Band cells, which leads to temporary throughput degradation.

In EN-DC deployments, the following limitations apply:

- If the EUtranCellFDD.measGapPattEndc or the EUtranCellTDD.measGapPattEndc attribute is set to 0, the following events might occur: In case of NR Low-Band serving cells, the gap placement might overlap with the NR TRS, which leads to throughput degradation. In case of NR Mid-Band serving cells, the gap placement might overlap with the periodic SRS, which leads to throughput degradation.
- SFN shifts between LTE and NR are supported, but it is recommended to avoid them. SFN shift is considered only at EN-DC setup for a given UE.
- The Master Node informs the Secondary Node that the maximum number of inter-frequency measurements to configure for the Secondary Node is set to 5. The Master Node does not check whether the limit is followed by the Secondary Node.
- If the UE, the Master Node, or Secondary Node does not support independent measurement gap configuration, a UE-level measurement gap is configured for all frequency ranges.

Network Requirements

The recommended parameter configurations for network synchronization must be used.

For more information on the recommended parameter configurations, see Network Synchronization Solution Guideline.

Any ESS cells for which the value of NRCellDU.ssbOffset is set to

1 and the value of NRCellDU.ssbDuration is

set to 1, inter-frequency measurements in EN-DC must be

reconfigured. To avoid measurement failures related to PSCell changes from the ESS

cell to a non-ESS cell, the value of the

GUtranSyncSignalFrequency.smtcOffset attribute must be set to

0.

To support independent measurement gap configuration, the UE must be capable of

reporting independent measurement gap configuration according to 3GPP Release

15.

To perform gapless inter-frequency measurements on High-Band frequencies, the UE must

be capable of gapless measurements.

To support gap patterns with a 3 ms measurement gap, the UE must be capable of

reporting 3 ms measurement gap support according to 3GPP Release 16.

In NR SA, the UE must support reporting the measurement gap requirement information

dynamically to the gNodeB for UE capability-specific measurement gap configuration

to work.

# 3 Feature Operation of Measurement Gap-Aware NR Scheduling

This feature works automatically when the configured measurements require a

measurement gap.

In EN-DC and NR-DC, the measurement gap configuration must be coordinated between the

Master Node and the Secondary Node in the following way:

- The Master Node must be able to configure the measurement gap when the Secondary Node requires inter-frequency measurements. Inter-frequency measurements that are configured by the Secondary Node are only supported in EN-DC.
- The Secondary Node must be able to apply the measurement gap that is configured by the Master Node.

Applying the measurement gap has the following effects:

- In EN-DC and NR-DC, UEs are not scheduled in downlink or uplink on the MCG and SCG radio resources during the measurement gap.
- In NR standalone, UEs are not scheduled during the measurement gap in downlink or uplink.

Measurements use a 3 ms measurement gap if the following

conditions are all met:

- The GNBDUFunction.measGapLengthMode attribute is set to a value that enables a 3 ms measurement gap for the measurement.
- The UE indicates support for 3 ms measurement gaps in one of the following UE capabilities: The supportedGapPattern UE capability which can be checked for LTE, and NR Low-Band and Mid-Band measurements. The supportedGapPatterns-NROnly-r16 UE capability which can be checked for NR Low-Band and Mid-Band measurements.
- A 3 ms measurement gap completely covers the SMTC window specified in all of the measurement objects. Measurement gaps are placed according to the SMTC offset specified in NR measurement objects. If the measurement gap does not completely cover the SMTC window, the SMTC configuration is recommended to be adjusted through the following attributes: NRFrequency.smtcDuration NRFrequency.smtcOffset
- The placement of the 3 ms measurement gap length allows the UE to measure LTE synchronization signals for LTE measurements.
- The SFN of the source cell aligns with the SFN of the target cells for LTE measurements. Note: If the SFN of the source cell and the SFN of the target cells are not aligned, the GNBDUFunction.measGapLengthMode attribute is recommended to be set to one of the following values: NR\_FR1\_3MS OFF

If the measurement cannot be configured with a 3 ms measurement gap

length, a measurement gap pattern that has a 6 ms measurement gap length is

used instead. The measurement gap repetition period of the measurement is

not affected by this change.

## 3.1 Measurement Gap Configuration and Coordination in EN-DC

### 3.1.1 Master Node Initiated Measurement Gap Configuration

If a measurement requires a measurement gap,

measurement

gap coordination can be done for EN-DC-configured UEs during the Master

Node Initiated Secondary Node Modification procedure.

1. When a measurement gap is configured, changed, or removed for a UE configured with EN-DC on the Master Node that supports independent measurement gap configuration, the Master Node applies one of the following measurement gaps: Independent measurement gap on Low-Band or Mid-Band frequencies if the UE and the Secondary Node support independent measurement gap configuration. A UE-level measurement gap for all frequency ranges if the UE or the Secondary Node does not support independent measurement gap configuration.
2. Optionally, the Master Node sends the measurement gap configuration to the Secondary Node during the Master Node initiated Secondary Node Modification process for the following cases: If a measurement gap is configured for all frequencies. If a measurement gap is configured for Low-Band or Mid-Band, and the PSCell is on Low-Band or Mid-Band frequencies. The Secondary Node applies the configured UE-level measurement gap for all frequency ranges, or the independent measurement gap for Low-Band or Mid-Band frequencies if the PSCell is in a Low-Band or Mid-Band frequency range. In other cases, the measurement gap configuration is ignored. The Secondary Node acknowledges the modification request if the Secondary Node Modification procedure is triggered. In the acknowledgment message, the Secondary Node might include SCG configuration updates if required, for example, for DRX or scheduling request resource configuration.
3. The Master Node configures the UE with the applicable measurement gap through the RRC RRCConnectionReconfiguration message.
4. If the optional Master Node initiated Secondary Node Modification process is necessary, the Master Node confirms the modification to the Secondary Node.

Figure 1   Master Node Initiated Secondary Node Modification

### 3.1.2 Secondary Node Initiated Measurement Gap Configuration

A measurement gap can be configured, changed, or removed for each EN-DC-configured UE on

the Secondary Node that supports independent measurement gap configuration.

1. The Secondary Node initiates the Secondary Node Modification procedure for the following purposes: To transfer the frequency information of the measurement on Low-Band or Mid-Band frequencies to the Master Node. Note: When a measurement gap is configured for all frequency ranges, High-Band frequencies are also transferred to the Master Node. This happens if the UE, the Master Node, or the Secondary Node does not support independent measurement gap configuration. To configure independent measurement gap for an inter-frequency measurement for the UE on High-Band frequencies.
2. The Master Node that supports independent measurement gap configuration applies the independent measurement gap configuration on Low-Band or Mid-Band NR frequencies if the UE also supports independent measurement gap configuration. The Master Node applies a UE-level measurement gap configuration for all frequencies if the UE does not support independent measurement gap configuration.
3. Optionally, the Master Node sends the measurement gap configuration to the Secondary Node during the nested Master Node Initiated Secondary Node Modification procedure in the following cases: If a measurement gap is configured for all frequencies. If a measurement gap is configured for Low-Band or Mid-Band, and the PSCell is in Low-Band or Mid-Band frequencies. The Secondary Node applies the configured UE-level measurement gap for all frequencies or the independent measurement gap for Low-Band or Mid-Band frequencies if the PSCell is in a Low-Band or Mid-Band frequency range. The Secondary Node responds with the X2AP MODIFICATION REQUEST ACKNOWLEDGE message to the modification request if the nested Master Node Initiated Secondary Node Modification procedure is triggered.
4. The Master Node uses the RRC Connection Reconfiguration procedure to configure the UE with one of the following measurement gaps: For UEs that support independent measurement gap configuration, independent measurement gap is configured for Low-Band or Mid-Band frequencies. For UEs that do not support independent gap configuration, the measurement gap is configured for all frequencies. The Master Node also transfers the independent measurement gap configuration to the UE on High-Band frequencies.
5. The Master Node sends the X2AP SGNB MODIFICATION CONFIRM message to the Secondary Node.

## 3.2 Measurement Gap Configuration in NR Standalone

The

measurement

gap can be configured for UEs in NR standalone deployments for

measurements that require it.

If the GNBCUCPFunction.nrNeedForGapsSupported attribute is set to

true and the UE supports measurement gap requirement information

reporting, measurement gaps can be configured dynamically. This follows the following

process:

1. The UE indicates measurement gap requirement information reporting capability during the UE Capability Enquiry procedure to the gNodeB.
2. The gNodeB initiates the reporting by sending a list of NR bands for which the UE has to send measurement gap requirement information in the following messages: The list contains the NR bands configured in the network that the UE also supports.
    - RRCReconfiguration
    - RRCResume
3. The UE starts reporting if it requires a measurement gap or not when performing inter-frequency measurements on each NR band received using the following messages: The information contains if the UE requires a measurement gap or not when performing inter-frequency measurements on each NR band received in the list from the gNodeB.
    - RRCReconfigurationComplete
    - RRCResumeComplete
4. The reported information is used by the gNodeB when it configures an inter-frequency measurement for the UE.

The UE can update the measurement gap requirement information during RRC Reconfiguration

procedures. If the information is updated, the gNodeB can perform one more RRC

Reconfiguration procedure to update the measurement gap configuration according to the

new information reported.

For UEs that support independent measurement gap configuration in an NR standalone

deployment, the

measurement

gap can be configured for inter-frequency measurements on Low-Band or

Mid-Band frequencies, or IRAT measurements that require a

measurement

gap.

The gNodeB applies one of the following measurement gap configurations if the measurement

gap is required:

- An independent measurement gap configuration is applied if the inter-frequency measurements contain Low-Band or Mid-Band frequencies, and the UE supports independent measurement gap configuration. No measurement gap is needed for High-Band inter-frequency measurements if the UE supports independent measurement gap configuration.
- A UE-level measurement gap configuration is applied for all frequencies if the inter-frequency measurements contain Low-Band or Mid-Band frequencies and the UE does not support independent measurement gap configuration.
- A UE-level measurement gap configuration is applied for High-Band frequencies if the UE does not support independent measurement gap configuration and the value of the CapabilityHandlingUeCfg.gaplessFr2InterFreqMeasSupp attribute is set to false. Note: No measurement gap is needed if the measurements only contain High-Band inter-frequency measurements and the value of the CapabilityHandlingUeCfg.gaplessFr2InterFreqMeasSupp attribute is set to true. Such measurements include A4 measurements for NR-DC setup and MDT-specific inter-frequency measurements on High-Band frequencies.

## 3.3 Measurement Gap Configuration and Coordination for the UE Configured with NR-DC

Measurement gap coordination for NR-DC-configured UEs can be done during the Master

Node initiated Secondary Node Modification, the Secondary Node Addition, and the Secondary

Node initiated Secondary Node Change NR-DC procedures.

As specified by 3GPP, measurement gap requirement information reporting is not supported

in NR-DC. When the UE is configured for NR-DC the measurement gap requirement

information reporting is stopped, and information provided by the UE is not used to

configure measurement gaps for NR-DC. During the Secondary Node Release procedure

reporting can be restarted, if needed.

When a measurement gap is configured, changed, or removed for a UE configured with NR-DC

by the Master Node, the Master Node applies one of the following types of measurement

gap configurations:

- Independent measurement gap is applied on Low-Band or Mid-Band frequencies if the UE supports independent measurement gap configuration. Note: Inter-frequency measurements on High-Band frequencies are not configured in NR-DC. Such measurements can be kept in NR-DC if these are started in NR standalone. In this case, the independent measurement gap can be configured for the UE on High-Band frequencies in NR-DC.
- Measurement gap is applied for all frequency ranges if the UE does not support independent measurement gap configuration.

When the measurement gap is added, changed, or removed by the Master Node on all

frequencies or on High-Band frequencies, the measurement gap is transferred to the

Secondary Node during the following procedures:

- Secondary Node Addition
- At the following Master Node-based handover procedures: Intra-gNodeB handover at Secondary Node Addition with immediate NR-DC setup Intra-cell handover at Secondary Node Addition with immediate NR-DC setup Inter-gNodeB handover at Secondary Node Addition with immediate NR-DC setup
- Secondary Node initiated Secondary Node Change
- Master Node initiated Secondary Node Modification

Figure 3   Secondary Node Addition and Secondary Node Initiated Secondary Node Change in

NR-DC

1. If the Secondary Node initiated Secondary Node Change procedure is triggered, the source Secondary Node sends the XnAP S-NODE CHANGE REQUIRED message to the Master Node.
2. In the XnAP S-NODE ADDITION REQUEST message, the Master Node forwards the measurement gap for all frequency ranges to the Secondary Node.
3. The target Secondary Node acknowledges the modification request, which contains SCG configuration updates.

# 4 Network Impact of Measurement Gap-Aware NR Scheduling

The feature has an impact on capacity and performance, and it affects multiple

interfaces.

Capacity and Performance

UE throughput is affected by measurement gaps. When measurement gaps are configured,

none of the slots that overlap with the measurement gaps can be used for scheduling

data to the UE.

UEs that support measurement gap requirement reporting can be configured with fewer

measurement gaps. The number of configured measurement gaps remains the same, if the

UE does not provide measurement gap requirements other than what is configured in

the network.

Despite reporting no gap requirement for a certain band, some UEs can still apply a

short gap to adjust the receiver. During this gap the UE cannot send or receive

data, which can cause an increase in the DTX rate.

Additional slots are blocked in downlink and uplink. In downlink, some of the slots

before the measurement gap can also be blocked. This occurs if the HARQ ACK or NACK

feedback transmission falls in the measurement gap or in the slot immediately after

the measurement gap. In uplink, some of the slots before the measurement gap can

also be blocked. This occurs if the uplink scheduling grant falls in the measurement

gap or in the slot immediately after the measurement gap.

The throughput degradation of measurements can be approximated with the ratio of the

measurement gap length and the measurement gap repetition period. The throughput

degradation depends on the following properties:

- Duplexing mode
- TDD pattern
- Starting location of the gap relative to the TDD pattern

The configuration of measurement gaps affects measurement latency. Different gap

patterns and periodicity affect the measurement time of the UE.

# 8 Engineering Guidelines for Measurement Gap-Aware NR Scheduling

## 8.1 System Frame Number Shift in EN-DC

Once the feature is operable, the number of DTXs on the NR

leg is reduced.

SFN shift between the MCG and SCG resources is supported. The

SFN shift is configured in the eNodeB using the following

attributes:

- EUtranCellFDD.frameStartOffset
- EUtranCellTDD.frameStartOffset
- ExternalGUtranCell.absSubFrameOffset
- ExternalGUtranCell.absTimeOffset

Based on these attributes, the eNodeB creates an

SFTD measurement report and forwards it to the gNodeB during Secondary Node Addition using the

measResultCellListSFTD-NR IE. The gNodeB

uses the first element in the list, irrespective of the PCI, to compute the SFN shift,

and applies the shift to any offset received from the eNodeB.

## 8.2 Feature Specifics for EN-DC and NR-DC with SCG in High-Band

The feature enables the configuration of lower values for the

DrxProfileUeCfg.drxOnDurationTimer and the

DrxProfileUeCfg.drxInactivityTimer attributes.

For the DrxProfileUeCfg.drxOnDurationTimer attribute, the

configuration of a shorter DRX on-duration timer to one of the following values is

recommended:

- ONDURATIONTIMER\_2MS
- ONDURATIONTIMER\_8MS

For the DrxProfileUeCfg.drxInactivityTimer attribute, the

configuration of a shorter DRX inactivity timer to one of the following values is

recommended:

- INACTIVITYTIMER\_8MS
- ONDURATIONTIMER\_10MS

For EN-DC, the Master Node supports the configuration of the following measurement gap

patterns:

- 0
- 1

For NR-DC, the Master Node supports the configuration of the following measurement gap

patterns:

- 0
- 1
- 2
- 3
- 14

## 8.3 Feature Specifics for EN-DC, NR Low-Band, and NR Mid-Band

During the measurement gap configuration process, the LTE configured gap can overlap with

the drxOnDurationTimer. If that happens, the timer is extended to

compensate for the missing slots during the measurement gap configuration process. If

the gap placement restricts or overlaps with a scheduling request, the scheduling

request is reconfigured to avoid an overlap.

In NR Low-Band and NR Mid-Band deployments, a measurement gap might overlap with the NR

TRS or the periodic SRS, respectively. If the NR TRS or the SRS has a 40 ms periodicity,

the following attributes must be set to a value other than 0:

- EUtranCellFDD.measGapPattEndc
- EUtranCellTDD.measGapPattEndc

If the values of these MO attributes are set to 0, the measurement

gap might completely overlap with the NR TRS or the periodic SRS.

## 8.4 Feature Specifics for NR Standalone

The gNodeB determines the gap placement based on the following factors:

- The period and the offset of the DRX cycle associated with UEs in connected mode
- The period and the offset of the scheduling request
- The period and the offset of the TRS
- The configuration of the NRFrequency.smtcOffset attribute

In NR Low-Band and NR Mid-Band deployments, if the NR TRS or the periodic SRS has a 40 ms

periodicity, measurement gaps for EPS fallback measurements might completely overlap

with the NR TRS or the periodic SRS, respectively, which leads to temporary throughput

degradation. The length of time that this overlap lasts can be reduced by modifying the

value of the McfbCellProfileUeCfg.epsFbTargetSearchTimer

attribute.

## 8.5 Reconfigure the SMTC Offset of ESS Cells in EN-DC

In EN-DC, the eNodeB determines the placement of the

measurement gap for the LTE measurements and the NR measurements configured by the gNodeB. The placement is based on the value of the

GUtranSyncSignalFrequency.smtcOffset MO attribute for the

PSCell.

If the PSCell is an ESS cell and the

GUtranSyncSignalFrequency.smtcOffset attribute is not set to

0, some of the SSB beams in the target NR frequencies might fall outside of the

measurement window. This is because the UE uses the first 0.5 milliseconds of the

measurement gap to tune into but not to measure on the target frequency.

Steps

1. Set the value of the GUtranSyncSignalFrequency.smtcOffset MO attribute to 0 without modifying the configuration of the gNodeB.

## 8.6 Configuring Inter-Frequency Measurements without Measurement Gap on High-Band Frequencies

Inter-frequency measurements can be configured without measurement gap on High-Band

frequencies in NR standalone deployments by using the

CapabilityHandlingUeCfg.gaplessFr2InterFreqMeasSupp

attribute.

By configuring the CapabilityHandlingUeCfg.gaplessFr2InterFreqMeasSupp

attribute, the

measurement

gap can be disabled for UEs that support gapless High-Band

inter-frequency measurements but do not report the support of the independent

measurement gap capability.

By setting the CapabilityHandlingUeCfg.gaplessFr2InterFreqMeasSupp

attribute to true, no measurement gap is configured. If the UE does not

report the support of the independent measurement gap capability, the measurement gap is

configured for the UE by setting the

CapabilityHandlingUeCfg.gaplessFr2InterFreqMeasSupp attribute to

false.

The CapabilityHandlingUeCfg.gaplessFr2InterFreqMeasSupp attribute can

be configured for different UE groups. The value of the

CapabilityHandlingUeCfg.gaplessFr2InterFreqMeasSupp attribute is

read from the MO instance that matches the UE group that the UE belongs to. This way,

for example, different

configurations

can be used for UEs that support independent measurement gap configuration and for UEs

that do not.

Summary and Benefits

This feature enhancement introduces the possibility to configure measurement gaps to

be 3 ms for the following measurements:

- Low-Band NR inter-frequency measurements
- Mid-Band NR inter-frequency measurements
- IRAT measurements

This feature enhancement also allows the measurement gap repetition period to be

configured to 40 ms for EPS fallback measurements used by the following features:

- EPS Fallback for IMS Voice
- NR Emergency Fallback to LTE

This feature enhancement has the following benefits:

- A 3 ms measurement gap might decrease throughput degradation compared to 6 ms measurement gaps.
- A 40 ms measurement gap repetition period allows for faster reporting and a shorter setup time for calls compared to an 80 ms measurement gap repetition period.
- Placing the measurement gap according to the SMTC offset of the target NR frequencies allows measurements of NR frequencies with System Frame Number (SFN) shift.

Capacity and Performance

The impact of this feature enhancement can be observed through the following EBS-N

counters of the NRCellDU MO class:

- pmEbsnMacDrbThpDlMbbHighVolDistr
- pmEbsnMacDrbThpDlMbbLowVolDistr
- pmEbsnMacUeThpUlMbbHighVolDistr
- pmEbsnMacUeThpUlMbbLowVolDistr

The impact of this feature enhancement can be observed through the

NRCellDU.pmMeasGapDurationDistr PM counter.

Operation

This feature enhancement introduces the

GNBDUFunction.measGapLengthMode attribute, which can be used

to specify a preferred measurement gap length for the supported measurements.

A 40 ms measurement gap repetition period is automatically enabled for all

measurements that support it.

This feature enhancement modifies the following PM counters:

- NRCellDU.pmMeasGapUeMaxDistr
- NRCellDU.pmMeasGapUeMnMaxDistr
- NRCellDU.pmMeasGapUeMnSumDistr
- NRCellDU.pmMeasGapUeSumDistr

Interfaces

No impact.

Hardware

This feature enhancement is only applicable in Low-Band and Mid-Band deployments.

This feature enhancement is supported on the RAN Compute units of the following RAN

Compute groups:

- RAN Compute group 4
- RAN Compute group 5a, 5b, 5c, and 5d
- RAN Compute group 6

Other Network Elements

It is required to use ENM version 25.2 or later to support the

MOM changes introduced by this feature enhancement.
