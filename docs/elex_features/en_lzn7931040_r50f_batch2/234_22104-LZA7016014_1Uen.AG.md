# 1 LTE Baseband Reset Overview

The LTE Baseband Reset basic feature introduces a restart domain. When a

software-related problem at baseband is detected, the Baseband is reset. This allows faster

recovery of the LTE functionality in the MSRBS.

| Feature Name           | LTE Baseband Reset                                     |
|------------------------|--------------------------------------------------------|
| Feature Identity       | FAJ 121 3091                                           |
| Value Package Name     | LTE Base Package                                       |
| Value Package Identity | FAJ 801 0400                                           |
| Node Type              | Baseband Radio Node                                    |
| Access Type            | LTE                                                    |
| Licensing              | Non-license-controlled feature. No license is required |

Summary

The feature reduces system downtime for a baseband fault from up to 4 minutes to

about 20 seconds by avoiding a node restart.

1. First, an LTE Baseband reset is initiated.
2. If the LTE Baseband reset is not successful, an LTE RAT restart is performed.
3. If this process fails too, a full node restart is initiated.

LTE Baseband reset can be prevented in case of software-related problems, to allow

continuous processing. In these cases, all corresponding UE context data are removed

from the eNodeB and MME. If the UE context release is not sufficient to fix the

corrupted data, LTE Baseband reset is performed.

Additional Information

More information about this feature, and related topics, is available in the

following documentation:

- Manage Faults
- Manage Software
- Manage Hardware Equipment

# 2 Dependencies of LTE Baseband Reset

The LTE Baseband Reset feature has no dependencies to other features.

Feature Dependencies

There are no dependencies to other features.

Hardware

This feature is not supported on AIR 6454 units.

Limitations

In mixed mode configurations on Radio Gateway R308 and R608 units, and LC-05 9xCPRI

line card, it is not possible to restart one single RAT. Therefore, a Baseband reset

on one RAT affects the other RAT or RATs as well, which results in a complete Radio

Gateway or LC-05 9xCPRI line card reset. In such a case, all cells for all RATs are

disabled.

Network Requirements

This feature has no prerequisite network configuration requirements.

# 3 Feature Operation of LTE Baseband Reset

Faster restart of baseband-related software problems is solved by resetting the baseband

software, that allows faster recovery of the LTE functionality in the MSRBS.

Baseband reset is a fast recovery method for baseband faults resulting in a short traffic

interruption. All UEs are lost and the impact on the ongoing traffic is registered in

the PM counters. Affected cells and cell down-time are logged in the availability

log.

During baseband reset, the following actions are performed:

- The event is logged in the availability log.
- The PMD with the relevant trace and error log from the central processor module program is created.
- Digital signal processor core dumps are stored on the disk.
- The core dump from the real-time core application is included as a file in the PMD.
- The restart impacts PM counters related to ongoing traffic.

The log contains records from program crashes, snapshots, and PMD data without a program

crash with fault information.

1. First, an LTE Baseband reset is initiated.
2. If the LTE Baseband reset is not successful, an LTE RAT restart is performed.
3. If this process fails too, a full node restart is initiated.

When the baseband resets, several UE level procedures fail and trigger PM Events.

# 4 Network Impact of LTE Baseband Reset

The LTE Baseband Reset feature produces a fast recovery from baseband faults resulting

in a shorter interrupt on traffic.

Capacity and Performance

The feature improves the end-user UE availability.

Interfaces

No impact.

Other Network Elements

No impact.

# 7 Activate Non-License-Controlled Features

Non-license-controlled features do not require manual activation and cannot be

deactivated.

While license-controlled features need to be activated, non-license-controlled features

are activated automatically during node integration. They also do not

have

a software license key associated, which means that they have no

entries in the LKF that is installed on the node.

In some cases, non-license-controlled features need to be configured

manually or have some of

their

subfunctions enabled to work properly.
